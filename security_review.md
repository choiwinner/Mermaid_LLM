# 기업 보안 검토 보고서 (Ollama 로컬 LLM 전환 시)

본 보고서는 Gemini API 대신 **Ollama를 통한 로컬 LLM(사내 호스팅)**으로 전환할 경우의 보안 적합성을 검토합니다. 결론적으로, Ollama 전환은 **기업 내 정보 보안 관점에서 볼 때 외부 API 방식보다 훨씬 안전하며 적합한 선택**입니다.

## 1. 데이터 보안 및 주권 (Data Sovereignty) - [매우 우수]
- **데이터 유출 차단**: 모든 대화 내용과 시스템 프롬프트가 외부(Google 등)로 전송되지 않고 사내 서버 내에서만 처리됩니다. 이는 기업 기밀이나 개인정보 유출 위험을 원천적으로 차단합니다.
- **폐쇄망 운영 가능**: 인터넷 연결 없이도 동작하므로, 강력한 보안이 요구되는 폐쇄망(Air-gapped) 환경에서도 구축 및 운영이 가능합니다.

## 2. 인증 및 프라이버시 관리 (Authentication & Privacy)
- **API 키 제거**: 외부 서비스용 API 키(GEMINI_API_KEY)를 관리할 필요가 없으므로, 키 유출로 인한 금전적 손실이나 오남용 위험이 사라집니다.
- **로컬 세션 관리**: Streamlit의 세션 상태는 브라우저와 서버 메모리 상에만 존재합니다. Ollama 서버 역시 로컬 통신을 하므로 중간 탈취(MITM) 위험이 현저히 낮습니다.

## 3. 네트워크 및 인프라 보안 (Infrastructure Security)
- **Ollama 포트 격리**: Ollama의 기본 포트(11434)가 사외로 노출되지 않도록 방화벽 설정을 하고, Streamlit 서버와 Ollama 서버 간에는 루프백(localhost) 또는 내부 전용망 통신만 허용해야 합니다.
- **모델 무결성**: `ollama pull`을 통해 모델을 가져올 때 신뢰할 수 있는 소스인지 확인하고, 사내에서 검증된 모델 가중치(Weights) 파일만 사용하도록 가이드라인을 수립해야 합니다.

## 4. 애플리케이션 보안 개선 사항 (필수 조치 완료)
- **CDN 제거 완료**: 이미 조치된 바와 같이, Mermaid.js 라이브러리를 로컬화([mermaid.min.js](file:///c:/python/Mermaid_LLM/packages/mermaid/dist/mermaid.min.js))하여 외부망 접속을 완전히 차단했으므로 오프라인 환경에서도 안전하게 시각화 기능을 사용할 수 있습니다.

## 5. 추가 권장 사항
- **입력값 필터링**: 보안 기능이 강화된 모델(예: Llama-3-Guard)을 병행 배치하여, 악의적인 질문이나 부적절한 데이터 입력 시도를 사전에 차단하는 것이 좋습니다.
- **감사 로그(Audit Log)**: 기업 내에서 사용 시, 어떤 사용자가 어떤 성격의 질문을 했는지에 대한 기록을 별도의 보안 로그로 남겨 사후 검토가 가능하도록 구성할 수 있습니다.

## 총평
Ollama 로컬 LLM 기반의 이 프로젝트는 **데이터 외부 유출이 불가능한 구조**로 설계 가능하므로, 기업 내 보안 가이드라인을 충족하기에 매우 유리합니다. 인프라의 물리적/논리적 격리만 잘 이루어진다면 **정보 보안 관점에서 상용 API 서비스보다 훨씬 높은 신뢰도**를 가질 수 있습니다.
